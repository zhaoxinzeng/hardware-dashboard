# 多硬件生态监控看板运营 Skill

## 适用场景

当用户已经有一个多硬件生态看板项目或演示工作区，并希望在**不脱离现有项目结构**的前提下持续维护内容时，使用本 Skill。

典型场景包括：

- 更新硬件伙伴新闻列表，并区分自动新闻与手动新闻
- 基于现有案例数据维护生态案例、行业标签和硬件标签
- 维护模型/硬件适配矩阵中的状态与展示文案
- 更新课程、活动、产品和用户反馈等看板演示数据
- 检查新闻抓取脚本、环境变量和定时工作流是否与当前展示需求一致
- 为演示版看板准备一套结构统一、可直接展示的静态数据

## 边界约束

- 本 Skill 只处理**工作区中真实存在**的能力、页面、脚本和数据结构。
- 如果仓库里只有说明文档，没有脚本、页面或工作流实现，则只能输出流程建议，不能假定这些能力已可直接执行。
- 如果用户提出的新需求超出当前项目能力，必须明确标记为“新增需求”或“未来迭代项”，不能伪装成现有能力。
- 公开仓库场景下不得写入密钥、账号密码或个人身份信息。

## 输入要求

- **项目路径或仓库上下文**：用于确认真实文件结构
- **本次维护目标**：新闻 / 案例 / 兼容矩阵 / 课程 / 活动 / 产品 / 反馈 / 演示数据联调
- **数据来源**：已有 JSON、页面表单录入、外部链接清单、人工整理结果等
- **关键约束**：是否允许改代码、是否允许改静态数据、是否需要保留演示默认值、是否需要兼容现有 LocalStorage 键名

## 执行步骤

### Step 1：先确认真实能力边界

1. 读取项目入口、路由、数据文件、脚本和工作流。
2. 列出当前仓库中已经存在的模块，例如：新闻、课程、活动、产品、反馈、案例、适配矩阵、外部工具嵌入。
3. 标记哪些能力是真实已实现的，哪些只是 README 或需求里提到但未落地的内容。

### Step 2：识别当前数据契约

按项目中的真实定义确认以下信息：

- 新闻字段：`id`、`date`、`title`、`imageUrl`、`summary`、`vendor`、`link`、`isManual`、`sourceType`
- 案例字段：`title`、`description`、`industry`、`hardware`、`url`、`isPinned`、`createdAt`
- 兼容矩阵结构：`models[]`、`hardwares[]`、`matrix{}`
- 本地存储键名、默认数据、置顶规则、预览条数限制

如发现字段缺失、命名冲突或 README 与代码不一致，优先以代码为准，并把差异记录在交付说明中。

### Step 3：按任务类型执行维护

#### A. 新闻维护

当任务是更新新闻模块时：

1. 先区分自动新闻与手动新闻的数据来源。
2. 若项目内已有抓取脚本（如 `scripts/fetch-news.js`），先检查数据源、关键词、AI 清洗逻辑、输出路径和环境变量名。
3. 仅在项目内真实存在对应脚本与配置时，才描述“可运行抓取与清洗”；否则只整理静态新闻数据。
4. 写入新闻时确保标题、日期、来源、链接、摘要、封面字段完整，避免破坏前端预览卡片。
5. 如自动摘要失败，应保留明确的降级说明，避免伪造结论。

#### B. 案例与兼容矩阵维护

当任务是维护案例或硬件适配信息时：

1. 使用现有案例字段，不擅自增加前端未消费的新字段。
2. 行业、硬件名称与推荐场景应尽量复用现有枚举和展示文案。
3. 兼容矩阵状态必须收敛到当前前端已支持的状态值，例如 `已跑通`、`适配中`、`规划中`。
4. 无法确认的适配状态，不得主观编造，应先标注待确认再更新。

#### C. 课程 / 活动 / 产品 / 反馈维护

1. 沿用当前项目已有的 CRUD 结构与预览逻辑。
2. 若首页只展示固定条数，更新数据时要兼顾“首页预览”和“子页完整列表”的一致性。
3. 若表单存在占位链接、默认值或置顶逻辑，应保留这些约束，避免因为数据清洗而破坏交互。

### Step 4：输出最小且可验证的改动

1. 只修改本次任务真正需要的文件。
2. 优先改已有 JSON、TS 类型、Hook 或页面文案，不做无依据的大范围重构。
3. 若改动涉及脚本或工作流，需同步说明依赖的环境变量和触发方式。
4. 若项目支持构建或 lint，交付前应执行最基础验证；若无法执行，也要明确说明原因。

## 输出要求

交付时至少包含以下内容：

1. **变更摘要**：本次更新了哪些模块、哪些文件、哪些数据
2. **能力边界说明**：哪些内容来自现有实现，哪些只是建议未落地
3. **验证结果**：构建 / lint / 手动检查 / 未验证原因
4. **待确认事项**：仍需业务方补充的数据、链接、状态或口径
